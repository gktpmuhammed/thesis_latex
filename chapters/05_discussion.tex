\chapter{Discussion}\label{chapter:discussion}

This chapter interprets the evidence from Chapter~\ref{chapter:experiments}. The goal is to explain why the observed ranking emerged, which conclusions are robust, and where claims should be bounded.

\section{Why Interface Design Dominated}
Empirical signal. Within MedGemma ablations, the strongest model, Prefix-Stable-1T, improves core global metrics such as GREEN, RadGraph, BLEU-4, ROUGE-L, BERTScore, and RadCliQ over Baseline-8T. By contrast, heavier variants show mixed or negative deltas despite higher nominal capacity.
For this pair, the gain should be read as a bundled token-interface effect because token count (8T vs 1T) and prefix design both change.

Mechanistic interpretation. Three interface components in this bundled setup appear decisive:
\begin{enumerate}
    \item explicit visual token delimitation (sentinel wrapping),
    \item consistent positional encoding over visual prefix tokens,
    \item stabilized projection into decoder embedding scale.
\end{enumerate}
Together they reduce ambiguity between visual context and textual instruction/response tokens. Since token count also changes in the strongest pair, these mechanisms are interpreted as plausible contributors rather than cleanly isolated factors.

Negative ablations in context. Perceiver-style resampling and multiscale fusion are not intrinsically weak. Under the current low effective batch, long context, and short training budget, their additional complexity likely increases optimization burden more than it improves report-level factuality.

Curriculum and hard-example strategies also underperform the best interface variant. This indicates that weighting/sampling can refine learning but cannot fully compensate for an unstable vision-language interface.

Case-study support. Cross-attention overlays provide spatial evidence consistent with metric trends. When a model exhibits diffuse or anatomically implausible attention despite larger capacity, degraded factual metrics are easier to explain. This is why case studies are treated as supporting evidence for mechanism, not only as qualitative illustration.

Design implication. For this task family, the practical order is to stabilize the visual-language interface first, verify organ grounding, and then scale architectural complexity.

\section{Metric Disagreement and Interpretation}
Different metric families reward different behavior, so disagreement is expected:
\begin{itemize}
    \item lexical metrics reward phrasing overlap,
    \item embedding metrics reward semantic proximity,
    \item clinical metrics reward factual and relational correctness.
\end{itemize}

The decoder results illustrate this directly. BERT-base is strongest on GREEN and broad lexical-semantic quality, while MedGemma is strongest on RadGraph. GPT-2 remains stronger than BioBART on most global dimensions. CheXbert remains useful but is not treated as the sole clinical proxy for organ-decomposed CT text. Consistent with Section~\ref{sec:evaluation_metrics}, model ranking prioritizes GREEN and RadGraph, then checks lexical-semantic coherence.

Diversity diagnostics add an important correction to aggregate metric ranking. In Table~\ref{tab:decoder_diversity}, BERT-base has high template reuse, with almost half of reports covered by its top fifty normalized templates. MedGemma shows substantially lower reuse. This explains why BERT can score well on global overlap-style metrics while still producing less case-specific report content.

\section{Generalization and Organ Coverage}
The chest-CT setting yields strong thoracic support and sparse support for several upper-abdominal organs. This long-tail distribution affects both training dynamics and evaluation variance:
\begin{itemize}
    \item sparse organs have noisier metrics,
    \item frequent thoracic patterns dominate global optimization,
    \item template bias risk increases for low-support organs.
\end{itemize}
Accordingly, organ-level conclusions are most reliable for high-support organs and should be interpreted cautiously for very sparse ones.

\section{Decoder Family Axis}
The decoder-comparison protocol is structurally fixed with a shared frontend, shared metric pipeline, and shared artifact policy. All target rows are now complete under this protocol: MedGemma, fVLM-aligned BERT-base, GPT-2, and BioBART. The current evidence favors BERT-base on overall global quality, MedGemma on relational-clinical structure by RadGraph, and GPT-2 over BioBART among the remaining decoder alternatives.

For practical deployment under organ-aware reporting objectives, MedGemma is the safer default in this thesis context because it combines stronger RadGraph performance with lower template repetition.

\section{Relation to Prior Paradigms}
Compared with many 2D report-generation settings, 3D CT introduces higher token pressure and stronger grounding demands. The results suggest that decoder scaling or heavier visual modules alone are insufficient if token semantics at the interface are not explicit and stable. This aligns with anatomy-guided trends in recent segmentation-aware and decomposition-aware literature.

\section{Validity and Limits}
Internal validity is strongest for MedGemma-family comparisons because data interfaces and evaluation pipelines are controlled. Decoder-family conclusions are now grounded by completed MedGemma, fVLM-aligned BERT-base, GPT-2, and BioBART rows under one protocol.
For RQ1 specifically, strict causal attribution to interface stabilization alone is limited because the top comparison jointly changes token budget and prefix structure.

Findings are specific to organ-decomposed chest-CT reporting and should not be over-generalized to other modalities or report styles without controlled replication.

Construct validity is strengthened by metric bundles plus case studies, but still depends on imperfect automatic metrics. Human expert evaluation remains outside the current chapter scope.

\section{Practical Implications and Synthesis}
For practitioners, the evidence supports an interface-first recipe:
\begin{enumerate}
    \item use explicit organ-conditioned supervision,
    \item stabilize visual prefix semantics and projection scale,
    \item run lightweight ablations before heavy architectural scaling,
    \item use metric bundles plus qualitative spatial audits,
    \item separate protocol-matched internal comparisons from external contextual baselines.
\end{enumerate}

Overall synthesis: under realistic compute constraints, interface calibration is the dominant success factor in this organ-aware 3D CT report-generation setup.
