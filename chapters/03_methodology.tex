\chapter{Methodology}\label{chapter:methodology}

\section{Pipeline Overview}\label{sec:method_overview}
This thesis studies organ-aware report generation from 3D chest CT. The model input is a CT volume
$V \in \mathbb{R}^{D \times H \times W}$ together with organ masks
$\{M_o\}_{o \in \mathcal{O}}$ for a fixed organ set $\mathcal{O}$. The output is an organ-structured report in which each organ receives a dedicated text segment.

For patient $i$, we model:
\begin{equation}
\hat{Y}_i = f_\theta\!\left(V_i, \{M_{i,o}\}_{o \in \mathcal{O}}\right),
\end{equation}
where $\hat{Y}_i = \{\hat{y}_{i,o}\}_{o \in \mathcal{O}}$ denotes organ-level generated text. A report-level prediction is assembled by concatenating organ sections in a fixed anatomical order.

The end-to-end workflow contains six stages:
\begin{enumerate}
    \item CT preprocessing and mask preparation.
    \item Report decomposition into organ-level supervision.
    \item Organ-aware visual token extraction from a 3D ViT encoder.
    \item Projection/bridging into decoder embedding space.
    \item Organ-conditioned text generation.
    \item Report assembly and metric computation.
\end{enumerate}

The same interface is used across four decoder families: MedGemma~\cite{yang2024medgemma}, BERT-base~\cite{devlin2019bert}, GPT-2~\cite{radford2019language}, and BioBART~\cite{yuan2022biobart}.

\section{Data and Targets}\label{sec:method_data_targets}
\subsection{Volume and Mask Preprocessing}
The training and evaluation code uses a consistent MONAI-based transform stack. Given raw NIfTI CT and segmentation volumes, preprocessing applies:
\begin{enumerate}
    \item channel standardization and axis transposition,
    \item HU clipping ($[-1150, 350]$) and min-max normalization to $[0,1]$,
    \item spatial harmonization to $(112,256,352)$ via padding/resizing,
    \item trilinear interpolation for CT and nearest-neighbor interpolation for masks.
\end{enumerate}

Replacing center-cropping with full-volume resizing reduces accidental exclusion of peripheral anatomy.

\subsection{Target Organ Set and Segmentation Mapping}
The final schema uses 12 organs:
\begin{equation}
\mathcal{O} = \{\text{lung, heart, esophagus, liver, gallbladder, stomach, pancreas, spleen, kidney, aorta, trachea, rib}\}.
\end{equation}

Mask extraction follows fixed segmentation-ID mappings summarized in Table~\ref{tab:organ_id_mapping}.

\subsection{Report Decomposition Targets}
Reference reports are decomposed into organ-keyed JSON records using an instruction-following LLM pipeline (Llama-3.3-70B-Instruct via vLLM~\cite{kwon2023efficient}, with AWQ quantization~\cite{lin2023awq}). The decomposition enforces:
\begin{enumerate}
    \item schema-constrained JSON output,
    \item one text field per organ key,
    \item explicit empty/null handling for missing organs.
\end{enumerate}

This converts free-text supervision into organ-conditioned targets and enables organ-level error analysis.

During training, each organ is paired with an organ-specific prompt. MedGemma branches use chat-format prompts; BERT-base/GPT-2/BioBART branches use shorter textual prompts (e.g., \texttt{Describe [organ]:}). In all branches, loss is masked to the response region only.

\subsection{Chest-CT Visibility Caveat}
Although the dataset modality is chest CT, several upper-abdominal organs appear in a non-trivial subset of scans. Visibility remains inconsistent across patients and protocols, so sparse organs are treated as optional supervision targets rather than mandatory output constraints.

\section{Model Architecture}\label{sec:method_architecture}
\subsection{3D Visual Backbone}
All principal variants use a 3D ViT encoder initialized from pretrained weights and fine-tuned end-to-end. The standard configuration uses input size $(112,256,352)$, patch size $(16,16,32)$, and hidden size 768.

\subsection{Organ Queries and Masked Cross-Attention}
Let $F(V) \in \mathbb{R}^{T \times d_v}$ be ViT token features and $Q_o \in \mathbb{R}^{K \times d_v}$ be learned queries for organ $o$. Organ masks are downsampled to the ViT token grid and converted into additive attention bias $B_o$ that blocks background tokens.

Organ token extraction is:
\begin{equation}
Z_o = \mathrm{MHA}(Q_o, F(V), F(V); B_o),
\end{equation}
where $Z_o \in \mathbb{R}^{K \times d_v}$. In implementation, mask downsampling is aligned to the token-grid geometry $(7,16,11)$.

\subsection{Token Granularity and Empty-Organ Safety}
Two token-interface regimes are used:
\begin{itemize}
    \item \textbf{Multi-token interface:} $K=8$ tokens per organ (96 tokens for 12 organs).
    \item \textbf{Compressed interface:} effective one-token-per-organ representation before decoder conditioning.
\end{itemize}

Empty masks are explicitly handled. If an organ query sees only blocked positions, attention bias is reset for that path to avoid undefined softmax behavior and NaN propagation.

\subsection{Projection, Stabilization, and Sentinel Wrapping}
Visual features are projected from $d_v=768$ to decoder hidden size $d_l$:
\begin{equation}
\tilde{Z}_o = W_p Z_o + b_p, \quad W_p \in \mathbb{R}^{d_v \times d_l}.
\end{equation}

Before concatenation with text embeddings, two stabilization steps are applied:
\begin{enumerate}
    \item layer normalization on projected visual tokens,
    \item variance matching to decoder embedding scale.
\end{enumerate}

For MedGemma, the sequence format is:
\begin{equation}
[\text{BOS}],\;\texttt{<vis>},\;\tilde{Z}_o,\;\texttt{<end\_vis>},\;\text{text prompt and target tokens}.
\end{equation}
Loss is masked over injected visual tokens so gradients are concentrated on target text generation.

\section{Decoder Adaptation and Training Objective}\label{sec:method_decoder_objective}
\subsection{Decoder-Specific Adaptation}
MedGemma is adapted with LoRA~\cite{hu2022lora} while the base model remains frozen. Main settings are rank $r=16$, $\alpha=32$, dropout $0.05$, and target modules
\texttt{q\_proj, k\_proj, v\_proj, o\_proj, gate\_proj, up\_proj, down\_proj}.

BERT-base, GPT-2, and BioBART branches use the same organ-attention visual frontend within a \texttt{VisionEncoderDecoderModel} interface. The BERT-base branch is trained with language modeling loss as a reproducible reference aligned with the fVLM decoder description. Decoder-side adaptation unfreezes a restricted subset (cross-attention/normalization/output layers and embeddings). For BioBART, shared embeddings and output-head weights are copied from pretrained checkpoints to avoid random-initialization drift in wrapper mode.

\subsection{Weighted Language Modeling Loss}
With batch index $i$, organ $o$, and token position $t$, the primary objective is:
\begin{equation}
\mathcal{L}_{\mathrm{LM}} = \frac{1}{B|\mathcal{O}|} \sum_{i=1}^{B}\sum_{o\in\mathcal{O}} w_{i,o}
\left(\frac{1}{T_{i,o}}\sum_{t=1}^{T_{i,o}} \mathrm{CE}(y_{i,o,t}, \hat{y}_{i,o,t})\right),
\end{equation}
where $w_{i,o}$ is an organ/sample weight for imbalance control.

\subsection{Optional Alignment Objective}
Some variants add visual-text contrastive alignment:
\begin{equation}
S_{o,o'} = \frac{\langle v_o, t_{o'} \rangle}{\tau},
\end{equation}
and optimize:
\begin{equation}
\mathcal{L} = \mathcal{L}_{\mathrm{LM}} + \lambda \mathcal{L}_{\mathrm{align}}.
\end{equation}
This is used mainly in \texttt{medgemma\_architecture\_v3} and selected BERT-base/GPT-2/BioBART branches.

\section{Optimization and Inference}\label{sec:method_optimization_inference}
\subsection{Shared Optimization Settings}
Across main retraining runs, settings are largely standardized:
\begin{itemize}
    \item batch size 1 per device, gradient accumulation 8,
    \item learning rate $1\times10^{-4}$, weight decay 0.01, warmup ratio 0.05,
    \item 2 retraining epochs,
    \item bf16 with gradient checkpointing,
    \item checkpoint/evaluation every 200 steps.
\end{itemize}

\subsection{Training-Strategy Variants}
Beyond architecture, the thesis evaluates curriculum learning, hard-example mining, undersampling of dominant ``no-finding'' targets, and organ-weighted losses.

\subsection{Organ-Wise Inference and Report Assembly}
At inference, one prompt is generated per organ and decoded with beam search. MedGemma runs typically use 3 beams and \texttt{max\_new\_tokens=100}; BERT-base/GPT-2/BioBART runs typically use 4 beams and \texttt{max\_length=120}. Outputs are assembled in fixed organ order:
\begin{equation}
\hat{R} = \texttt{LUNG: }\hat{y}_{\mathrm{lung}}\;\Vert\;\cdots\;\Vert\;\texttt{KIDNEY: }\hat{y}_{\mathrm{kidney}}.
\end{equation}
Organs with weak or absent visual support are not forced to produce abnormal statements, reducing schema-induced hallucination.

\section{Reproducibility Notes}\label{sec:method_reproducibility}
\begin{itemize}
    \item Training/evaluation artifacts are isolated per variant to prevent checkpoint and metric contamination.
    \item Official runs include generated report files and standardized \texttt{metrics\_final.csv} summaries.
    \item Metric computation is centralized through one RadEval-based script with fixed key mapping.
    \item Decoder comparison uses only official organ-attention result folders.
\end{itemize}

This design keeps the thesis contribution focused: controlling organ grounding and token interface under a consistent data protocol, then comparing architecture and decoder choices on that shared foundation.
