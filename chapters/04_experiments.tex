\chapter{Experiments and Results}\label{chapter:experiments}

\section{Experimental Setup}\label{sec:exp_setup}
\subsection{Objectives and Research Questions}
This chapter evaluates organ-aware 3D CT report generation under a consistent CT-RATE protocol. Results are organized by four research questions:
\begin{itemize}
    \item \textbf{RQ1 (Token Interface):} Does organ-token interface design improve report quality?
    \item \textbf{RQ2 (Architecture):} Do heavier visual modules outperform the simpler masked cross-attention baseline?
    \item \textbf{RQ3 (Training Strategy):} How much do curriculum/hard-example/imbalance policies change outcomes?
    \item \textbf{RQ4 (Decoder Family):} Under the same frontend, how do MedGemma, GPT-2, and BioBART compare?
\end{itemize}

\section{Evaluation Protocol}\label{sec:exp_eval_protocol}
\subsection{Metric Families}\label{sec:evaluation_metrics}
We report complementary metric groups: lexical overlap (BLEU-4~\cite{papineni2002bleu}, ROUGE-L~\cite{lin2004rouge}, METEOR~\cite{banerjee2005meteor}, CIDEr~\cite{vedantam2015cider}), semantic similarity (BERTScore~\cite{zhang2019bertscore}), and clinical/factual consistency (CheXbert~\cite{smit2020chexbert}, RadGraph~\cite{jain2021radgraph}, GREEN~\cite{ostmeier2024green}, RadCliQ~\cite{yu2023radcliq}, SRR-BERT style score).

Formal equations are provided in Appendix~\ref{appendix:metric_equations}. Chapter-level interpretation focuses on metric behavior and agreement patterns.

\subsection{Metric Interpretation Policy}
Model ranking is anchored on GREEN and RadGraph, then cross-checked with lexical-semantic metrics. CheXbert is reported but interpreted cautiously for organ-decomposed CT text because its design assumptions are closer to full-report label extraction workflows.

\subsection{Evaluation Pipeline and Artifact Standardization}
For each model checkpoint:
\begin{enumerate}
    \item Generate reports and export \texttt{generated\_reports.csv} (or \texttt{generated\_reports\_gemma.csv}).
    \item Compute metrics with unified \texttt{radeval\_metrics.py}.
    \item Export standardized \texttt{metrics\_final.csv} with GLOBAL and per-organ rows.
\end{enumerate}

The key mapping from RadEval output to thesis columns is fixed (e.g., \texttt{green} $\to$ GREEN, \texttt{radgraph\_simple} $\to$ RadGraph, \texttt{radcliq-v1} $\to$ RadCliQ), ensuring direct comparability across variants.

\subsection{Source-of-Truth Policy}
\begin{itemize}
    \item MedGemma ablations: \texttt{fvlm/rep\_medgemma/results\_retrain/global\_model\_comparison.csv}.
    \item Decoder comparison: official \texttt{rep\_vision\_organ\_attention/results/gpt2\_v3} and \texttt{.../biobart\_v3} artifacts.
    \item External baseline: fVLM paper statistics~\cite{shui2025fvlm} in a separate contextual block.
\end{itemize}

\subsection{Qualitative Protocol for Attention Maps}
Cross-attention maps are interpreted with the following checks: spatial overlap with the queried organ mask, compactness vs diffusion, stability across planes, and false activation in anatomically confounding neighbors.

\subsection{RQ3: Training Strategy}
Training-policy interventions (hard-example mining, undersampling, weighted-loss variants) are competitive but do not surpass the best interface variant. Curriculum learning degrades most global metrics in the current two-phase configuration.

\section{Chapter Summary and Minimal Limitations Note}\label{sec:exp_summary}
The strongest internal model under current protocol is \texttt{medgemma\_lora\_vis\_token\_pos\_embed}. The result pattern is consistent across RQ1--RQ3: interface stabilization is more beneficial than heavier architectural modules or training-policy complexity at the current compute budget.

The decoder axis (RQ4) remains a core comparison direction with finalized table structure and official artifacts. As of March 3, 2026, MedGemma metrics are complete; GPT-2/BioBART rows are pending the final standardized metric pass from official v3 generated reports.

Interpretation limits in this chapter are concise: uneven organ support in chest CT, partial metric-domain mismatch for organ-decomposed text, and non-equivalent external baseline protocol for fVLM. A full validity discussion is provided in Chapter~\ref{chapter:discussion}.
